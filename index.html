Microsoft: Software Engineering Intern

During the summer of 2016, I was a Software Engineering Intern in Seattle, WA working on the Windows Servicing and Delivery team at Microsoft. Over the course of my internship, I built a cloud-based configuration system to move environment variable management from individual users and files checked into source control repositories to the Microsoft Cloud. Environment and configuration variables normally requiring extensive setup when creating new development environments (e.g. DEV, Pre-Production, Production) transitioned from taking roughly 2 hours to taking a matter of minutes. The configuration system was built using C# and Microsoft Azure Service Fabric and at peak request load served over 131,000 requests an hour. The cloud-based centralized configuration store is used to configure several parts of the windows update pipeline, including OnePub, the part of the pipeline that publishes update bundles to Windows users.

Parallelizing Git Rename Detection

As part of an operating systems class at Vanderbilt, students are challenged to either choose a final project from several options or to make and submit a proposal for a custom project. Samuel Lijin, a good friend of mine, and I decided to take on the challenge of optimizing parts of git, a popular open source version control system. Our proposal included parallelizing both diffs and rename detection for when files are changed but also renamed between commits in git.

The full report can be found here along with the challenges associated with these tasks and the conclusions made. Essentially, running on two cores, we found that parallelizing rename detection performed roughly the same as in the current single threaded model but that locking overhead made it difficult to jet past single core counterparts.
